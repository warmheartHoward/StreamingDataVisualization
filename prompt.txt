请帮我开发一个基于 Python **Streamlit** 的可视化项目，要求代码采用模块化结构。

### 1. 目录结构
- `app.py`: 主程序，负责 UI 布局、键盘事件处理与状态更新。
- `src/parser.py`: 负责解析 JSON 文件，特别需要提取 `frame_path` 字段（这是一个包含图片路径的 list）。
- `src/__init__.py`: 模块标识。

### 2. 详细功能需求

#### 区块 A：左上 (配置区)
- **数据路径**: `st.text_input` 获取根目录。
- **最大同时展示数目**: `st.number_input` 限制右侧数据组的总数。

#### 区块 B：左下 (目录交互区)
- **主文件夹展示栏**: 顶部显示一个文本框，缺省值 "XXX"。
- **右键双击交互**: 通过 JS 监听目录树，右键双击某个文件时，自动将其名称填入“主文件夹展示栏”。
- **右键触发逻辑**：请利用自定义 HTML/JS 监听右键点击事件，或提供一个显眼的“开启选择模式”按钮。右键点击后，目录进入“可选中状态”。
- **左键选择逻辑**：在可选中状态下，用户点击文件夹进行勾选。选中文件夹后，触发右侧 C 和 D 区块的更新。
- **约束检查**：
    - 实时监测已勾选的数量。
    - **非法操作拦截**：若勾选总数超过区块 A 设定的“最大同时展示数目”，立即弹出 `st.toast` 或 `st.error` 报警提示“超过最大展示限制”，并自动取消当前违规的勾选。

#### 区块 C：右上 (JSON 展示区)
- 使用 `src/parser.py` 解析选中文件夹下的 JSON。
- 提取并展示 JSON 内容，确保能读取到 `frame_path` 字段。

#### 区块 D：右下 (图片序列交互展示区) —— **核心逻辑**
- **数据来源**: 来自 JSON 中的 `frame_path` 列表。
- **展示逻辑 (滑动窗口)**:
    1. **初始化**: 初始只展示序列中的第 1 张图。
    2. **窗口增长**: 
       - 按下键盘 **右方向键 (Right Arrow)**：若当前展示不足 3 张，则在右侧新增下一张；若已展示 3 张，则整体向右滚动（删除最左侧旧图，右侧新增下一张）。
    3. **窗口回退**: 
       - 按下键盘 **左方向键 (Left Arrow)**：窗口向左滚动（右侧删除一张，左侧新增上一张上一张）。
    4. **约束**: 任何时刻画面中最多只展示 3 张图片。
- **交互实现**: 必须通过 JavaScript 监听全局 `keydown` 事件（`ArrowLeft`, `ArrowRight`），并通过 `window.parent.postMessage` 将按键信号传回 Streamlit 的 `session_state`。

### 3. 技术规范
1. **src/parser.py**: 编写 `get_frame_paths(json_path)` 函数，返回图片路径列表。
2. **状态管理**: 在 `st.session_state` 中维护一个 `display_indices` 列表（存储当前显示的图片索引）。
3. **JS 注入**: 
   - 编写 JS 监听 `contextmenu` 实现右键双击功能。
   - 编写 JS 监听 `keydown` 实现左右键翻页功能。
4. **Mock 数据**: 提供一个包含 10 个虚拟图片路径的 JSON 示例，确保在没有真实图片时也能演示按键滚动效果。

请输出完整的工程代码。
